## ТОН ГОЛОСА (Блок 4)

**Архетип:** 70% Мудрец (данные, факты, экспертность) + 30% Друг-советник (тёплый, доступный, человечный)

**Мы звучим:** экспертно, дружелюбно, уверенно, просто, прямо
**Мы НЕ звучим:** формально-бюрократически, агрессивно-продажно, заискивающе, безлично

### Словарь — ИСПОЛЬЗУЕМ:
- Аналитик, эксперт (не «менеджер», не «агент»)
- Инвестиционный консалтинг (не «продажа недвижимости»)
- Доходность, ROI, кейс (не «скидка», «акция»)
- Подбор под вашу задачу (не «каталог объектов»)

### Словарь — НЕ ИСПОЛЬЗУЕМ:
- «Элитная недвижимость», «тропический рай», «уникальное предложение»
- «Поторопитесь», «осталось 2 квартиры» (давление)
- «Риелтор», «агент», «менеджер по продажам»
- «Недорого», «бюджетно»

### Тон по каналам:
| Канал | Тон |
|-------|-----|
| Авито | Конкретный + экспертный (аналитик, а не перекупщик) |
| Prian / Директ | Экспертно-образовательный |
| Instagram RU | Тёплый + экспертный (жизнь, а не продажа) |
| Instagram EN | Professional + data-driven + aspirational |
| Telegram | Инсайдерский + аналитический |
| Зумы / встречи | Аналитик + друг (сначала слушаем клиента) |

## ПРАВИЛА РАБОТЫ В ЭТОМ ПРОЕКТЕ

1. **Формат файлов:** Каждый документ = PDF/PPTX (для людей) + MD-дубликат (для AI-контекста)
2. **Структура папок:** Строго по схеме выше — не класть файлы в корень без необходимости
3. **Обновление саммари:** После каждой сессии обновлять `00_PROJECT_SUMMARY.md`
4. **Без одобрения:** Работать без остановок на стандартные шаги
5. **Визуальные примеры:** Включать в документы готовые примеры, а не абстрактные инструкции
6. **Именование:** snake_case для файлов, с префиксами по уровню (B1_, 03_, etc.)
